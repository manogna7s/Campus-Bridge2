Authentication Troubleshooting

If you're experiencing "Internal Server Error" when trying to register or login, follow these steps to diagnose and fix the issue.

1 Check Database Connection

First, verify that your database is properly connected:

  • Visit the Database Status Page
  • Click "Check Connection" to test database connectivity
  • If it fails, check your environment variables in Render

2 Verify Environment Variables

Ensure these environment variables are set in your Render dashboard:

MYSQL_HOST=your-database-host
MYSQL_PORT=3306
MYSQL_USER=your-database-username
MYSQL_PASSWORD=your-database-password
MYSQL_DATABASE=lms

For email functionality (optional but recommended):

EMAIL_SERVICE=gmail
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password
EMAIL_FROM=your-email@gmail.com
APP_URL=https://your-app-name.onrender.com

3 Check Database Tables

Ensure the required tables exist in your database:

CREATE TABLE IF NOT EXISTS learning_resources (
  id INT AUTO_INCREMENT PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  file_path VARCHAR(255) NOT NULL
);

CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  is_verified TINYINT(1) DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

4 Check Render Logs

Examine the application logs for detailed error information:

  1. Go to your Render dashboard
  2. Navigate to your web service
  3. Click on the "Logs" tab
  4. Look for error messages when you try to register or login

5 Common Error Solutions

ER_ACCESS_DENIED_ERROR
Your database username or password is incorrect. Double-check your environment variables.
ENOTFOUND
Your database host is incorrect or not accessible. Verify your database connection details.
ER_BAD_DB_ERROR
Your database name is incorrect. Make sure MYSQL_DATABASE is set to "lms".
TIP: If you don't need email verification during development, you can temporarily remove the is_verified check in the login endpoint by modifying the query to:
SELECT * FROM users WHERE email = ? AND password = ?
